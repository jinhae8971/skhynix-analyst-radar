# Memory Analyst Radar (SK Hynix · Samsung · Micron)

메모리 3사 — **SK하이닉스(000660)·삼성전자(005930)·마이크론(MU)** — 애널리스트 **투자의견·목표주가 상향/하향 추세** 자동 트래킹 시스템. 하나의 대시보드에서 **종목 탭**으로 전환해 비교합니다.

- **데이터 소스**
  - SK하이닉스·삼성전자 : 한경 컨센서스(consensus.hankyung.com) 국내 증권사 기업 리포트 (원화)
  - 마이크론 : Yahoo Finance 미국 애널리스트 등급/목표가 변경 이력 (달러) — 한경 컨센서스에는 미국 상장 마이크론 커버리지가 없어 별도 소스 사용
- **분석**: 리서치사별 최신 컨센서스, 목표가 리비전(상향▲/하향▼), 의견 변경, 30/90일 모멘텀, 유효/휴면 커버리지 분리. 종목 통화(₩/$)에 맞춰 자동 표기
- **원문 인리치(한경 종목 한정)**: 신규 리포트만 원문 PDF를 조회해 발표 시점 현재주가·상승여력(정형 수치)을 부착. `ANTHROPIC_API_KEY` 설정 시, 각 리포트의 논지를 저작권 안전하게 **재서술한 한 줄 요약**을 생성해 모달에 표시(미설정 시 제목만). 원문 문장은 저장하지 않음. (마이크론은 원문 PDF가 없어 등급/목표가 이력만 제공)
- **분산 시각화**: 트렌드 차트에 컨센서스 90일 이동평균 + **±1σ 분산 밴드**(진한) + **최저~최고 범위**(옅은)를 이중 밴드로 표현
- **상세 모달**: 리서치사 클릭 시 목표가·의견 변경 이력을 모달로 확인. 한경 종목은 상승여력·논지·요약·원문 링크까지, 마이크론은 등급/목표가 변경 이력을 표시
- **산출물**
  - `dashboard.html` / `index.html` — Chart.js 탭 대시보드 (GitHub Pages 공개 가능)
  - 텔레그램 정기 브리핑 (전략 비서 채널, 종목별 신규 리포트 발생 시 발송)
  - `state.json` — 종목별 신규 리포트 감지 상태 (워크플로우가 자동 커밋)
  - `summaries.json` — report_idx별 수치·요약 캐시 (신규만 PDF 조회, 워크플로우가 자동 커밋)

## 배포 (Windows)
```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\setup_github.ps1
```
레포 생성 → 푸시 → Secrets 등록 → Pages 활성화 → 최초 실행까지 자동.

## 로컬 테스트
```bash
pip install -r requirements.txt
cp config.json.example config.json   # 토큰 입력
python skhynix_ratings.py            # DASHBOARD_ONLY=1 이면 텔레그램 생략
```

## 스케줄 (GitHub Actions)
평일 KST 08:00 · 16:30 자동 실행. 신규 리포트가 없으면 발송하지 않음(중복 방지).
스케줄 변경은 `.github/workflows/skhynix_ratings.yml`의 cron 수정.

## 커버리지 기준
최근 180일 내 리포트를 발간한 증권사만 '유효 컨센서스'로 집계하고, 그 이전은 '휴면'으로 분리 표기.
투자판단의 책임은 이용자 본인에게 있습니다.

## 참고
- 최초 실행은 전체 리포트 원문(PDF)을 조회하므로 수 분 소요됩니다. 이후에는 `summaries.json` 캐시로 신규 리포트만 조회해 빠릅니다. 동봉된 `summaries.json`을 그대로 두면 최초 실행도 즉시 완료됩니다.
- PDF 수치 추출을 끄려면 환경변수 `ENRICH=0`. 실행당 신규 조회 상한은 `MAX_ENRICH`(기본 120).
- 데이터 소스는 네이버 차단 이슈로 한경 컨센서스를 사용합니다.
